import helloTs from "/src/content/examples/hello.ts?raw";
import helloPy from "/src/content/examples/hello.py?raw";

## Dealing with code

### Basic level

to import code you first need to import the code component
just paste `import { Code } from "@astrojs/starlight/components";` in the first line after the front matter section

afterwards you need to import the code snippet such as like `import helloPy from "/src/content/examples/hello.py?raw";`

you can then use the code component like this `<Code code={helloPy} lang="py" title="hello.py" />`

and the result is the following

<Code code={helloPy} lang="py" title="hello.py" />

### Mid level

Given that we expect to have multiple languages for the same examples we might want to display them in tabs

to import tabs you first need to import the tab component like this `import { Tabs, TabItem } from '@astrojs/starlight/components';`

the structure to give the `.mdx` in the mdx file is the following

```mdx
<Tabs>
	<TabItem label="Stars" icon="star">
		import code snippet or what not
	</TabItem>
	<TabItem label="Moons" icon="moon">
		wow
	</TabItem>
</Tabs>
```

so if we put in there the code components this happens

<Tabs>
	<TabItem label="Python" icon="seti:python">
		<Code code={helloPy} lang="py" title="hello.py" />
	</TabItem>
	<TabItem label="TypeScript" icon="seti:typescript">
		<Code code={helloTs} lang="ts" title="hello.ts" />
	</TabItem>
</Tabs>

if your first reaction is `ewww why is there a header on the component after the tabbed bar`
I hear you, the `code` component exposes a quirky prop named `frame` so if we set frame to `none`
the following snippet

```mdx
<Tabs>
	<TabItem label="Python" icon="seti:python">
		<Code code={helloPy} lang="py" frame="none" />
	</TabItem>
	<TabItem label="TypeScript" icon="seti:typescript">
		<Code code={helloTs} lang="ts" frame="none" />
	</TabItem>
</Tabs>
```

gives us

<Tabs>
	<TabItem label="Python" icon="seti:python">
		<Code code={helloPy} lang="py" frame="none" />
	</TabItem>
	<TabItem label="TypeScript" icon="seti:typescript">
		<Code code={helloTs} lang="ts" frame="none" />
	</TabItem>
</Tabs>
### Goat level

As you may have noticed the tabs don't keep sync on the preferred language, luckily `Tabs` component expose a useful prop called `syncKey`

so if to the top we add `syncKey="lang"` it'll check the label of each `TabItem` and keep it in sync

```mdx
<Tabs syncKey="lang">
	<TabItem label="Python" icon="seti:python">
		<Code code={helloPy} lang="py" frame="none" />
	</TabItem>
	<TabItem label="TypeScript" icon="seti:typescript">
		<Code code={helloTs} lang="ts" frame="none" />
	</TabItem>
</Tabs>
```

the two following are synced

<Tabs syncKey="lang">
	<TabItem label="Python" icon="seti:python">
		<Code code={helloPy} lang="py" frame="none" />
	</TabItem>
	<TabItem label="TypeScript" icon="seti:typescript">
		<Code code={helloTs} lang="ts" frame="none" />
	</TabItem>
</Tabs>
<Tabs syncKey="lang">
	<TabItem label="Python" icon="seti:python">
		<Code code={helloPy} lang="py" frame="none" />
	</TabItem>
	<TabItem label="TypeScript" icon="seti:typescript">
		<Code code={helloTs} lang="ts" frame="none" />
	</TabItem>
</Tabs>
